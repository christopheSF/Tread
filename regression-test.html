Observations & Bugs Found
OBS-01 Observation Time picker reverts after combobox interaction
Affected: Create Order dialog — Order Start Time field

Repro:
  • Open Create Order dialog
  • Select Customer Account (combobox)
  • Select Associated Project (combobox)
  • Fill Order Start Time (e.g. "02:00 PM")
  • Click Create

Result: Time reverts to default 9:00 PM — the fill is ignored when it occurs after a combobox selection event.
Workaround: Set time before selecting project/customer, or edit after creation. V1–V7 times were set before project selection (Pass); V8–V10 set after (Obs).
Severity: Medium — affects UX for operators who fill form top-to-bottom.
OBS-02 Observation Combobox filter requires two-step fill to trigger search
Affected: Customer Account & Associated Project comboboxes

Repro:
  • Click into the Associated Project combobox
  • Type "Reg May18" → list shows all 30+ unfiltered results
  • Clear field and type "May18 V1" → correctly filtered list appears

Result: First fill on a fresh empty combobox does not trigger the API filter call. A second fill with a different value triggers filtering correctly.
Severity: Low — workaround is straightforward; no data loss.
