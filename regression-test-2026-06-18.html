Methodology & honest scope note

Tool: Chrome DevTools driven by Claude Code via the accessibility tree (snapshot → grep uid → click/fill). All actions through the UI; no direct API calls. Every create verified to its success toast and resulting list/board state.

Why depth varies: this is a single sequential browser session. Reaching 5 data variations on all 18 groups is ~90 form-heavy UI cases; that exceeds one automated session. T-P01 was taken to full depth (5 + negative + edge) to prove the pattern and exercise the shared form/combobox/validation components that every other create-flow reuses. Remaining groups are marked Pending honestly rather than asserted.

Recommended path to full 5× coverage: the Playwright e2e suite (yarn e2e:test) is the right vehicle and already targets staging — it is currently blocked by a local dependency-install issue (GITHUB_PACKAGES_PAT). Unblocking it enables the full matrix headlessly; this manual run is the interim substitute.
