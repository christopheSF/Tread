Known Bugs
BUG-PLR-001 Zero-value rate card silently discarded on save Medium
Phase: P1  |  Requirements: R-P1.3, R-P1.4
Steps:
  1. Open Update Project modal for PROJ-12649
  2. Add a rate card with Customer Rate = $0.00, no conditions
  3. Click Save
Expected: Card saved (zero is a valid rate); or validation error shown.
Actual: Card disappears silently — no error, no confirmation. Other cards unaffected.
Impact: Silent data loss; operator unaware the card was not saved.
BUG-PLR-002 Rate cards reorder after save (wrong sort order) Low
Phase: P1, P2  |  Requirements: R-P1.6, R-P2.8
Steps:
  1. Save multiple rate cards with varying numbers of conditions
  2. Observe card order in Update Project modal and Create Order form
Expected: Cards sorted least→most conditions (V1, V2, V3, V4).
Actual: Cards appear in server/alphabetical order: Full Spec Rate first.
Impact: UX inconsistency; operators expect logical progression from simple to complex.
BUG-PLR-003 Automation: combobox condition fields not reliably selectable Low / Automation
Phase: P1  |  Requirements: Test infrastructure
Description: During automated test execution, MUI combobox dropdowns for condition fields (Material, Equipment Type, Pickup Site, Dropoff Site) within the Update Project rate card form did not reliably open or accept typed input via devtools fill/click. Manual workarounds used.
Impact: Test automation only; no user-facing impact.
BUG-PLR-004 Timeline audit events omit condition names from rate card entries Medium
Phase: P5  |  Requirements: R-P5.16, R-P5.24
Steps:
  1. Create a project with rate cards that have conditions (Material, Equipment Type, etc.)
  2. Open project timeline / audit log
  3. Inspect "Freight Rates created" events
Expected: Each event shows rate card name, rate value(s), AND condition names.
Actual: Events show name and rate values but condition names are absent.
Impact: Audit trail incomplete; operators cannot determine what conditions were active at time of creation from the timeline alone.
