03

How it behaves in production

Feature-flagged

Nothing shows until the Split.io flag product_tours is on for a company. Rollout can be staged company-by-company.

Runs once

Seen state is stored per browser. Dismissing a tour counts as seen — users are never nagged twice.

Orientation first

The App Orientation tour runs on first login anywhere. Completing it chains straight into the current page's tour; dismissing it doesn't.

Replayable

"Take a quick tour" in the account menu re-runs the current page's tour any time.

Adapts to permissions

Steps anchored to elements a user can't see (e.g. New Order for view-only users, company switcher for single-company users) are dropped from their tour.

Waits for the page

The tour waits up to 8s for the page's anchors to render before starting, so slow loads never produce a popover pointing at nothing.
