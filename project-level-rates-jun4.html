New AUTO-1 Setup A: Create Add On dialog has 2 async-autocomplete inputs (strict mode violation)
Confirmed — June 4 manual investigation. Navigated to Settings → Rates → Add On Charges and opened "Create Add On". When Rate Type is set to Fuel Surcharge, the dialog renders two elements matching data-test-id="async-autocomplete": (1) a disabled Company autocomplete pre-filled with "Staging Hauler Co" and (2) the new Fuel Surcharge Schedule autocomplete that appears on Fuel Surcharge selection. Playwright strict mode fails because getByTestId('async-autocomplete') now resolves 2 elements. Root cause: a new disabled Company field was added to the Add On dialog. Fix: use a more specific selector (e.g. targeting the visible/enabled autocomplete) or add a unique data-test-id to the schedule picker.
New AUTO-2 Attach project add-on returns 400 in automated run
API regression confirmed — June 4 manual investigation. Calling POST /v1/projects/{id}/auto_add_on_assignments with the existing payload {"data":{"add_on_id":"…","account_type":"customer","quantity":1}} returns 400 Bad Request: "#/components/schemas/AutoAddOnAssignment-Create does not define properties: data". The API schema has changed: the body wrapper data: is no longer valid. A flat payload attempt returns a different 400: "missing required parameters: name, add_on_rate_type, add_on_types" — showing the endpoint now expects create-a-new-add-on fields rather than attach-an-existing-add-on fields. The auto_add_on_assignments schema was likely updated in the API spec since May 21. Fix: update attachProjectAddOn e2e flow to match the new schema.
Race Condition AUTO-3 Add-on edit button blocked by MuiDataGrid overlay — race condition, not a product bug
Not a product bug — confirmed June 4 manual investigation. Manually navigated to Settings → Rates → Add On Charges. When the page fully loads, the edit pencil button for every add-on is clickable and opens the edit dialog without issue. The automated failure occurs because the test clicks before the DataGrid finishes loading its data, while the MuiDataGrid-overlay (the empty-state or loading overlay) is still covering the grid. Fix: add a wait for the DataGrid to exit loading state before clicking the edit button (e.g. wait for overlay to be detached, or wait for at least one row to be visible).
